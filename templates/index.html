How it works, in normal human language

The project is built in layers so you can explain both the pretty front end and the science-y plumbing behind it.

01

Retrieval layer

You type a topic. BioLitGraph asks PubMed for the most relevant papers and grabs each paper’s basic metadata: title, journal, date, and PMID.

02

Annotation layer

Those PMIDs get sent to PubTator, which tags the biology inside the literature — like genes, diseases, chemicals, and variants — and tries to normalize them into standard IDs.

03

Graph layer

The backend collapses repeated mentions into one entity node, links entities that appear together, counts how many papers support each connection, and saves the results as tables plus graph JSON.

04

App layer

The browser turns that graph JSON into an interactive network. Click a node or edge and the details panel shows what it is, how connected it is, and which papers support it.
